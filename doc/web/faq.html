CLOPS FAQ

1. What is the difference between isFooSet() and getFoo() for a boolean option Foo?

Any option either is unset or is set and has a value: Options whose value is a boolean are no exception. If you want to know if a certain string (alias) appeared on the command line then you want to use isFooSet(). 2. Why does CLOPS use the special character \0 to separate strings?

It is customary that the shell process the command line and chops it up into a list of strings. In Java this is reflected by String[] args in the main method of any program. It is not true, however, that each of the strings in the string input list would correspond to exactly one options. One string may represent multiple options (e.g., ps aux) or multiple string may correspond to one option (e.g., head -n 999).

For this reason CLOPS turns the list of options given to the program by the shell into a single string where the original substrings are delimited by a special character \0. For instance the above example results in the string "-n\0999" As a result each option may freely specify to which sequence of characters or strings it corresponds. For the example above, we may choose the regular expression "-n\0[0-9]*". CLOPS, however, have mechanisms that let the user not deal with the special character for the most common cases, such as one-argument-option.
